# Quick Task 260329-m36 Summary

**Fix prisma/seed.ts to create hiring stages and assign candidate to first stage**

## Objective

Resolve the constraint violation `check_hiring_stage_when_job_assigned` by:
1. Creating 8 hiring stages for each of the 3 jobs (24 total)
2. Assigning Yael Cohen candidate to the first hiring stage
3. Assigning the Application record to the same first hiring stage

## Changes Made

### Constants Added

- Added 24 deterministic UUIDs for hiring stages across 3 jobs:
  - `STAGES_SE[0-7]`: 00000000-0000-0000-0000-000000002001 through 000000002008
  - `STAGES_PM[0-7]`: 00000000-0000-0000-0000-000000002009 through 000000002016

- `DEFAULT_HIRING_STAGES` array with 8 stages copied from `src/jobs/jobs.service.ts`:
  1. Application Review (order 1, bg-zinc-400, enabled)
  2. Screening (order 2, bg-blue-500, enabled)
  3. Interview (order 3, bg-indigo-400, enabled)
  4. Offer (order 4, bg-emerald-500, enabled)

  7. Pending Decision (order 7, bg-yellow-400, disabled)

### Job Creation Flow

Changed from `upsert` to `deleteMany` + `create` pattern to ensure nested `hiringStages` creation executes. For each job:
- Delete any existing job records (cascades to related hiring stages)
- Create job with nested `hiringStages.create` mapping DEFAULT_HIRING_STAGES to deterministic UUIDs
- Each job now has 8 hiring stages with proper tenant and stage IDs

### Candidate Assignment

Added `hiringStageId: FIRST_STAGE_SE` to Yael Cohen's candidate record, pointing to "Application Review" stage of the Senior Software Engineer job.

### Application Assignment

Added `jobStageId: FIRST_STAGE_SE` to the application record, connecting the application to the same first hiring stage.
