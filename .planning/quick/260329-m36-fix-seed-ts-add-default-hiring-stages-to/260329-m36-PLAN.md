## Problem Statement

The seed creates:
- 3 jobs WITHOUT hiring stages
- 1 candidate (Yael Cohen) with `jobId: JOB_SE` but `hiringStageId: null`

This violates the schema constraint at Candidate line 99-100:
```
@@index([tenantId, jobId, hiringStageId], name: "idx_candidates_tenant_job_stage")
```

And the raw SQL check: `CHECK(job_id IS NULL OR hiring_stage_id IS NOT NULL)`

## Solution

Update seed.ts to:
1. Create DEFAULT_HIRING_STAGES (8 stages) for each of the 3 jobs
2. Assign Yael Cohen to the first stage (Application Review) of JOB_SE
3. Assign the Application to the same first stage
