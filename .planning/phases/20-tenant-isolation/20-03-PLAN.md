# Plan 20-03: Wave 3 — Test Updates + Full Suite Verification + Smoke Test

## Goal

Update all affected unit and integration tests to match the new service signatures (tenantId as explicit param, ConfigService removed from services). Run the full test suite to green. Manual smoke test confirms two separate org users see only their org's data.

## Context

- Waves 1 and 2 changed controller + service signatures throughout candidates, jobs, and applications
- Tests mock `ConfigService` to return `TENANT_ID` — those mocks are now dead code since services no longer call `configService.get('TENANT_ID')`
- Service unit tests call service methods directly — call sites must be updated to pass `tenantId` param
- Controller spec files test endpoint routing — they mock the service, so those mock return values are unaffected but call assertion expectations may reference `configService`

## Files to Change

## Task Breakdown

### Task 1 — Update `candidates.service.spec.ts`

2. **Remove** `mockConfigService.get.mockReturnValue(TENANT_ID)` calls in beforeEach/test bodies
3. **Update** every direct service method call to pass `TENANT_ID` as the appropriate param:

```typescript

// Before

// After

// Before

// Before
```

### Task 8 — Manual Smoke Test (Human Checkpoint)

**Prereq:** Two orgs created via Phase 19 auth endpoints, each with at least one candidate/job.

**Steps:**

1. Sign in as User A (Org A) via `POST /auth/google/verify` → get `talent_os_session` cookie A
2. Sign in as User B (Org B) via `POST /auth/google/verify` → get `talent_os_session` cookie B
3. Verify `GET /candidates` with Cookie A returns only Org A candidates
4. Verify `GET /candidates` with Cookie B returns only Org B candidates
5. Verify `GET /jobs` with Cookie A returns only Org A jobs
6. Verify `GET /candidates` with no cookie returns `401 Unauthorized`
7. Verify `GET /config` with no cookie returns `401 Unauthorized`

**Expected:** No cross-tenant data leakage. All unauthenticated requests rejected.
