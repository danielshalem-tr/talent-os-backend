# Plan 20-01: Wave 1 â€” SessionGuard on All Business Controllers + env.ts Updates

## Goal

Add `@UseGuards(SessionGuard)` to every business endpoint (candidates, jobs, applications, config), extract `tenantId = req.session!.org` in each controller action, and pass it down to service methods. Also make `TENANT_ID` optional in `env.ts` (still required for IngestionProcessor; not required for API server).

## Context

- `SessionGuard` lives at `src/auth/session.guard.ts` â€” injects `request.session: JwtPayload` where `JwtPayload.org` = organization UUID
- `JwtPayload` is declared in `src/auth/jwt.service.ts`; Express `Request` is augmented there to accept `session?: JwtPayload`
- All business services currently call `this.configService.get<string>('TENANT_ID')!` internally â€” Wave 1 threads `tenantId` through the controller without yet changing the service signatures (that's Wave 2)
- Wave 1 uses a temporary bridge: controller extracts `tenantId`, passes it as an additional arg; services will be refactored in Wave 2

### Task 3 â€” Guard and thread tenantId through CandidatesController

File: `src/candidates/candidates.controller.ts`

Add imports:

```typescript
import { UseGuards, Req } from '@nestjs/common';
import { Request } from 'express';
import { SessionGuard } from '../auth/session.guard';
```

Add `@UseGuards(SessionGuard)` to class level (applies to all routes).

For each controller method, add `@Req() req: Request` param and extract:

```typescript
const tenantId = req.session!.org;
```

Then pass `tenantId` as the **last** argument to every service call. Since Wave 2 hasn't changed service signatures yet, Wave 1 uses a **temporary shim**: the service methods are called with an extra `tenantId` arg that will be used after Wave 2 refactor. To avoid TypeScript errors in Wave 1, pass tenantId directly â€” we'll update service signatures in Wave 2.

> **IMPORTANT:** Wave 1 + Wave 2 must be executed together in the same session. Wave 1 alone will cause TypeScript errors (service methods don't accept tenantId yet). Only run `tsc` and tests after Wave 2 is also complete.

Affected methods (add `@Req() req: Request` + extract tenantId):

- `getCounts` â†’ `this.candidatesService.getCounts(tenantId)` (after Wave 2)
- `findAll` â†’ `this.candidatesService.findAll(tenantId, q, filter, jobId, unassignedBool)`
- `findOne` â†’ `this.candidatesService.findOne(id, tenantId)`
- `getCvUrl` â†’ `this.candidatesService.getCvPresignedUrl(candidateId, tenantId)`
- `create` â†’ `this.candidatesService.createCandidate(result.data, file, tenantId)`
- `updateStage` â†’ `this.candidatesService.updateStage(id, result.data, tenantId)`
- `delete` â†’ `this.candidatesService.deleteCandidate(id, tenantId)`
- `updateCandidate` â†’ `this.candidatesService.updateCandidate(id, result.data, tenantId)`
