# Plan 20-02: Wave 2 — Service Method Signature Refactor (tenantId as Param)

## Goal

Replace every `this.configService.get<string>('TENANT_ID')!` call in `CandidatesService`, `JobsService`, and `ApplicationsService` with an explicit `tenantId: string` parameter. Remove `ConfigService` injection from these services entirely. Verify `DedupService` and `IngestionProcessor` are untouched.

## Context

- Wave 1 updated all controllers to extract `tenantId = req.session!.org` and pass it to service methods
- Wave 2 updates service signatures to accept `tenantId` — this is where the compiler errors from Wave 1 are resolved
- `CandidatesService` has 12 occurrences of `configService.get('TENANT_ID')`
- `JobsService` has 7 occurrences
- `ApplicationsService` has 1 occurrence
- Several service methods call other service methods internally — those internal calls must also thread tenantId through

## Files to Change

| File | Change |
|------|--------|
| `src/candidates/candidates.service.ts` | Add `tenantId: string` param to 11 public methods; remove ConfigService injection |
| `src/jobs/jobs.service.ts` | Add `tenantId: string` param to 7 methods; remove ConfigService injection |
| `src/applications/applications.service.ts` | Add `tenantId: string` param to `findAll`; remove ConfigService injection |
| `src/candidates/candidates.module.ts` | Remove `ConfigService` from providers (if it was only needed for TENANT_ID) |
| `src/jobs/jobs.module.ts` | Same |
| `src/applications/applications.module.ts` | Same |

> **Note:** Do NOT remove `ConfigService` from `IngestionModule`, `DedupModule`, `WebhooksModule`, or `StorageModule` — those modules legitimately use it for non-TENANT_ID config.

## Task Breakdown

### Task 1 — Refactor `CandidatesService`

**Method signature changes** (add `tenantId: string` as first param where service pulls it from config; for methods where it's an internal detail, add as last param to maintain readability):

```typescript
// Before
async getCounts(): Promise<...>
// After
async getCounts(tenantId: string): Promise<...>

// Before
async findAll(q?, filter?, jobId?, unassigned?): Promise<...>
// After
async findAll(tenantId: string, q?, filter?, jobId?, unassigned?): Promise<...>

// Before
async findOne(candidateId: string): Promise<CandidateResponse>
// After
async findOne(candidateId: string, tenantId: string): Promise<CandidateResponse>

// Before
async updateCandidate(candidateId: string, dto: UpdateCandidateDto): Promise<CandidateResponse>
// After
async updateCandidate(candidateId: string, dto: UpdateCandidateDto, tenantId: string): Promise<CandidateResponse>

// Before
async rejectCandidate(candidateId: string, dto: RejectCandidateDto): Promise<CandidateResponse>
// After
async rejectCandidate(candidateId: string, dto: RejectCandidateDto, tenantId: string): Promise<CandidateResponse>

// Before
async saveStageSummary(candidateId: string, stageId: string, summary: string): Promise<...>
// After
async saveStageSummary(candidateId: string, stageId: string, summary: string, tenantId: string): Promise<...>

// Before
async advanceWithSummary(candidateId: string, currentStageId: string, summary: string): Promise<...>
// After
async advanceWithSummary(candidateId: string, currentStageId: string, summary: string, tenantId: string): Promise<...>

// Before
async updateStage(id: string, dto: UpdateCandidateStageDto): Promise<...>
// After
async updateStage(id: string, dto: UpdateCandidateStageDto, tenantId: string): Promise<...>

// Before

// Before

// Before
```

**Internal method calls that must be updated:**

The following methods call other service methods internally — thread tenantId through:

- `updateCandidate` (lines ~300, ~390, ~446, ~458): calls `this.findOne(candidateId)` → change to `this.findOne(candidateId, tenantId)`
- `rejectCandidate` (line ~484): calls `this.findOne(candidateId)` → change to `this.findOne(candidateId, tenantId)`
- `advanceWithSummary` (line ~540): calls `this.saveStageSummary(candidateId, currentStageId, summary)` → change to `this.saveStageSummary(candidateId, currentStageId, summary, tenantId)`
- `advanceWithSummary` (line ~576): calls `this.updateStage(candidateId, ...)` → change to `this.updateStage(candidateId, ..., tenantId)`

**Remove ConfigService:**

1. Remove `ConfigService` from constructor injection
2. Remove `import { ConfigService } from '@nestjs/config'` if no longer needed
